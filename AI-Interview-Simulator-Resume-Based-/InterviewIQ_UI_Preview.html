The Problem

Most Developers Are
Unprepared for Interviews

Technical skills aren't enough. The gap between knowing how to code and knowing how to communicate your value costs candidates offers every day.

⏱️
60-Second First Impression
Recruiters spend under 90 seconds on initial screening calls. Candidates who can't articulate their experience clearly are filtered out before they even begin.
🎭
Generic Practice Doesn't Help
Common interview prep tools ask the same generic questions regardless of your background. Real interviews are tailored to your resume — your prep should be too.
📊
No Objective Feedback Loop
Practicing alone in front of a mirror gives zero signal. You need structured, scored feedback on every answer to know if you're actually improving.
🏗️
Projects Need Narratives
Developers can build impressive projects but struggle to explain their impact, trade-offs, and decision-making — the exact things interviewers probe for.
How It Works

Features

Everything You Need
to Prepare Seriously

🔍
Smart Resume Parsing
Extracts 50+ technology skills, project names, and experience from PDF and TXT resumes using pattern recognition and a curated skill database.
🎯
Personalised Questions
Every question is generated from your specific skills and projects — "Walk me through your {skill} project" not "Tell me about yourself."
⭐
Real-Time 1–5 Scoring
Answers are evaluated on depth, impact-oriented language, STAR structure signals, and quantified outcomes. Transparent, consistent rubric every time.
💡
Actionable Improvement Tips
Each feedback includes one specific, implementable tip — not generic advice. "Add a quantified outcome" beats "be more specific."
📊
Session Score Dashboard
End-of-session breakdown showing average score, question-by-question review, and category breakdown: Excellent / Good / Average / Weak.
🔀
Mixed-Category Flow
Questions are intelligently interleaved — Technical, Behavioural, Project — to simulate a real interview's rhythm and pacing, not a monotonous drill.
Code Improvements

Original vs Improved

Every module was refactored for production quality, separation of concerns, and real-world robustness.

❌ Original Code
🔴Regex used raw string escaping bug (\b not \\b) — skills never matched
🔴Only 11 hardcoded skills detected
🔴Project extraction fragile — only found "Project:" prefix
🔴Scoring only used word count — no quality signals
🔴All questions randomly shuffled — unrealistic flow
🔴Tips were generic, not answer-specific
🔴No answer word count tracked
🔴No score breakdown (excellent/good/weak)
✅ Improved Code
🟢Fixed regex escape — skills now correctly detected
🟢50+ skills across 8 categories with whole-word matching
🟢4 extraction patterns: bullets, headers, pipes, numbered lists
🟢Scoring uses length + 25 impact keywords + STAR signals + numbers
🟢Interleaved Tech → HR → Project for realistic interview pacing
🟢Tips target the specific gap (no numbers, no STAR, too brief)
🟢Word count shown in feedback for self-awareness
🟢Full breakdown: 5★/4★/3★/1-2★ count in summary
